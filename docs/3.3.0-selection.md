# 选区

> 描述编辑器中光标或选中范围的位置。选区基于文档的位置系统，随文档变化通过映射器更新。

## Selection trait

所有选区类型实现同一个 trait：

| 字段/方法 | 说明 |
|-----------|------|
| anchor | 锚点——选区的固定端（用户开始拖选的位置） |
| head | 活动端——选区的移动端（用户拖选到的位置） |
| from | 选区的较小位置（= min(anchor, head)） |
| to | 选区的较大位置（= max(anchor, head)） |
| empty | 选区是否为空（from == to，即光标） |
| selection.map(doc, mapping) → Selection | 通过映射器将选区映射到新文档中 |
| selection.content() → Slice | 获取选区内容的切片 |
| selection.eq(other) → bool | 相等比较 |

anchor 和 head 的区分很重要——用户从右往左拖选时，anchor 在右、head 在左，from/to 始终是 from < to。

## 选区类型

### 文本选区 (TextSelection)

最常见的选区类型。光标和文本范围选择都是它。

| 字段 | 说明 |
|------|------|
| anchor | 锚点位置（必须在行内内容中） |
| head | 活动端位置（必须在行内内容中） |

```
光标（空选区）:
  anchor = 5, head = 5
  from = 5, to = 5, empty = true

选中 "粗体" 二字（从左往右选）:
  anchor = 5, head = 7
  from = 5, to = 7, empty = false

选中 "粗体" 二字（从右往左选）:
  anchor = 7, head = 5
  from = 5, to = 7, empty = false
  （from/to 相同，但 anchor/head 反了——影响后续键盘操作的方向）
```

创建方法：

| 方法 | 说明 |
|------|------|
| TextSelection::create(doc, anchor, head) | 创建文本选区。head 可省略，默认 = anchor（光标）。 |
| TextSelection::between(doc, anchor, head, bias) | 创建文本选区，如果位置不在合法的行内位置，自动沿 bias 方向搜索最近的合法位置。 |

---

### 节点选区 (NodeSelection)

选中整个节点。通常通过 Ctrl+点击 或选中图片等叶子节点触发。

| 字段 | 说明 |
|------|------|
| node | 被选中的节点 |
| anchor / from | 节点之前的位置 |
| head / to | 节点之后的位置（= from + node.node_size） |

```
选中一个 horizontal_rule:
  from = 4（节点前）
  to = 5（节点后，因为叶子节点 node_size = 1）
  node = horizontal_rule

选中一个 heading(level=1, "标题"):
  from = 0（节点前）
  to = 4（节点后，因为 content.size=2 + 开闭标签 2 = node_size 4）
  node = heading
```

创建方法：

| 方法 | 说明 |
|------|------|
| NodeSelection::create(doc, pos) | pos 是节点之前的位置 |

---

### 间隙光标 (GapCursor)

当光标位于两个不允许放置文本选区的节点之间时使用。比如两个代码块之间、表格和段落之间——这些位置无法放置文本光标，但用户需要能把光标放在那里以便插入新内容。

| 字段 | 说明 |
|------|------|
| pos | 间隙位置（= anchor = head = from = to） |

```
两个 code_block 之间:

  code_block("...")
  ← 间隙光标可以放在这里（位置 N）
  code_block("...")

  这个位置不在任何 textblock 内部，TextSelection 不合法
  GapCursor(pos=N) 表示光标在这个间隙中
```

## 选区的映射

文档修改后选区位置可能失效，需要通过映射器更新：

```
selection.map(doc, mapping) → Selection
```

映射规则：
- anchor 用 bias=-1 映射（固定端尽量不动）
- head 用 bias=1 映射（活动端跟着内容走）
- 映射后如果位置不合法，自动查找最近的合法位置
- 如果映射后 anchor == head，文本选区变成光标

## 选区与视图的交互

视图层需要根据选区类型渲染不同的视觉效果：

| 选区类型 | 视觉表现 |
|----------|---------|
| 文本选区（光标） | 闪烁的竖线 |
| 文本选区（范围） | 选中文字的蓝色背景 |
| 节点选区 | 被选中节点的蓝色边框或背景 |
| 间隙光标 | 节点间隙中的竖线 |

在 Canvas + cosmic-text 方案下，这些都需要在渲染时自行绘制。
