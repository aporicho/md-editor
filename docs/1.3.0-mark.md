# 标记

> 叠加在文本节点上的行内样式。标记不是节点，不占位置，不影响文档结构，只描述文字的"外观"。

## 标记 vs 节点

为什么粗体不是一种节点？

```
节点方案（不采用）：
paragraph
├── text "一段"
├── bold                ← 嵌套一层
│   └── italic          ← 再嵌套一层
│       └── text "交叉"
└── text "文本"

标记方案（采用）：
paragraph
├── text "一段"
├── text "交叉" [bold, italic]   ← 平铺，标记叠加
└── text "文本"
```

标记方案的好处：
- **无歧义**：`bold > italic > text` 和 `italic > bold > text` 是同一个东西吗？标记方案不存在这个问题
- **操作简单**：添加/移除一个样式只需修改标记集，不需要拆分或合并节点层级
- **Markdown 天然适配**：`***交叉***` 就是同时有 bold 和 italic，没有嵌套顺序

---

## 标记类型

| 标记类型 | 属性 | 对应 Markdown |
|----------|------|---------------|
| bold | — | `**text**` |
| italic | — | `*text*` |
| code | — | `` `text` `` |
| link | href, title | `[text](href)` |
| strikethrough | — | `~~text~~` |

---

## 标记组成

| 字段 | 说明 |
|------|------|
| type | 标记类型（bold、italic 等），由约束器定义 |
| attrs | 属性键值对（如 link 的 href 和 title） |

两个标记相等 = 类型相同且属性相同。

---

## 标记集

文本节点上的标记是一个有序集合（`Vec<Mark>`），顺序由约束器决定，与用户输入顺序无关。

```
约束器定义的标记优先级: link > bold > italic > code > strikethrough

无论用户先加粗还是先斜体，标记集始终排列为:
[bold, italic]
```

固定顺序保证了相等比较的确定性——不会因为添加顺序不同导致两个本质相同的文本节点不相等。

---

## 关键操作

| 操作 | 说明 |
|------|------|
| mark.eq(other) | 类型和属性都相同 |
| mark.add_to_set(set) | 将标记加入标记集，按约束器顺序插入，同类型替换 |
| mark.remove_from_set(set) | 从标记集中移除同类型标记 |
| mark.is_in_set(set) | 检查标记集中是否存在同类型标记 |
| Mark::same_set(a, b) | 两个标记集是否完全相同（片段判断是否需要合并文本节点时使用） |
