# 编辑器状态

> 编辑器某一时刻的完整快照。不可变值，所有更新通过 apply(事务) 产生新实例。

## 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| doc | Node | 当前文档的节点树 |
| selection | Selection | 当前光标/选区 |
| stored_marks | Option\<Vec\<Mark\>\> | 暂存标记（None 表示没有暂存，使用光标处实际标记） |
| schema | Schema | 约束器引用（所有状态共享同一个约束器） |
| plugins | Vec\<Plugin\> | 插件列表（创建时确定，之后不变） |

---

## 创建

### 从零创建

```
EditorState::create(config):
  config.schema    — 约束器（必需）
  config.doc       — 初始文档（可选，默认创建包含一个空段落的文档）
  config.selection — 初始选区（可选，默认光标在文档开头）
  config.plugins   — 插件列表（可选）
```

### 从 Markdown 文件创建

```
let doc = parser.parse(markdown_string);
let state = EditorState::create(schema, doc, plugins);
```

---

## 核心方法

| 方法 | 说明 |
|------|------|
| state.tr() | 创建一个以当前状态为起点的事务。事务的初始文档 = state.doc，初始选区 = state.selection。 |
| state.apply(tr) → EditorState | 应用事务，返回新状态。 |
| state.apply_inner(tr) → (EditorState, bool) | 同 apply，额外返回选区是否发生了显式设置（插件判断用） |

---

## apply 详解

apply 是状态层的核心——它把事务转化为新状态：

```
state.apply(tr):

  // 1. 文档
  new_doc = tr.doc  // 事务中步骤应用后的最终文档

  // 2. 选区
  if tr 显式设置了选区:
    new_selection = tr 中设置的选区
  else:
    new_selection = state.selection.map(tr.doc, tr.mapping)
    // 把旧选区的位置通过映射器转换到新文档中

  // 3. 暂存标记
  if tr 显式设置了暂存标记:
    new_stored_marks = tr 中设置的暂存标记
  else if tr 有文档修改 or 选区显式变更:
    new_stored_marks = None  // 清除
  else:
    new_stored_marks = state.stored_marks  // 保持不变

  // 4. 插件状态
  for plugin in plugins:
    new_plugin_state[plugin] = plugin.apply(tr, state.plugin_state[plugin])

  return EditorState { new_doc, new_selection, new_stored_marks, ... }
```

---

## 插件状态的读取

每个插件通过插件键 (PluginKey) 标识，从状态中读取自己的状态：

```
let history_state = history_key.get_state(&state);
```

---

## 序列化

| 方法 | 说明 |
|------|------|
| state.to_json() | 序列化为 JSON（文档 + 选区 + 插件状态） |
| EditorState::from_json(schema, json, plugins) | 从 JSON 恢复状态 |

用于编辑器会话的保存和恢复（比如浏览器刷新后恢复编辑位置）。
