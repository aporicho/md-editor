# 输入规则器

> 监听文本输入，当输入匹配预定义模式时自动触发事务。实现 Markdown 风格的即时转换。

## 原理

用户每次输入字符后，输入规则器检查光标前的文本是否匹配某个正则表达式。匹配时自动生成并提交事务。

```
用户在段落开头输入 "## ":

  每输入一个字符:
    "#"  → 不匹配任何规则
    "##" → 不匹配
    "## " → 匹配规则: /^(#{1,6})\s$/
             → 触发事务: 将 paragraph 转换为 heading(level=2)，删除 "## "
```

---

## 输入规则结构

| 字段 | 说明 |
|------|------|
| match | 正则表达式，匹配光标所在行的文本（光标前的部分） |
| handler | 匹配后的处理函数，接收匹配结果和状态，返回事务 |

---

## Markdown 输入规则

### 块级转换

| 输入 | 匹配模式 | 转换结果 |
|------|----------|---------|
| `# ` ~ `###### ` | `^(#{1,6})\s$` | paragraph → heading(level=N) |
| `> ` | `^>\s$` | paragraph → blockquote 中的 paragraph |
| `- ` 或 `* ` | `^[-*]\s$` | paragraph → bullet_list 中的 list_item |
| `1. ` | `^\d+\.\s$` | paragraph → ordered_list 中的 list_item |
| ` ``` ` | `^```$` | paragraph → code_block |
| `---` | `^---$` | paragraph → horizontal_rule |

### 行内转换

| 输入 | 匹配模式 | 转换结果 |
|------|----------|---------|
| `**text**` | `\*\*(.+)\*\*$` | 将匹配文本添加 bold 标记 |
| `*text*` | `(?<!\*)\*(.+)\*$` | 将匹配文本添加 italic 标记 |
| `` `text` `` | `` `(.+)`$ `` | 将匹配文本添加 code 标记 |
| `~~text~~` | `~~(.+)~~$` | 将匹配文本添加 strikethrough 标记 |

## handler 函数

### 块级转换的 handler

```
// "## " → heading(level=2)
handler(state, match, start, end):
  let tr = state.tr()
  tr.delete(start, end)                              // 删除 "## "
  tr.set_node_markup(start_of_line, heading, {level: 2})  // 段落 → 标题
  return tr
```

### 行内转换的 handler

```
// "**text**" → text[bold]
handler(state, match, start, end):
  let tr = state.tr()
  let text_start = start + match.prefix_len  // "**" 的长度
  let text_end = end - match.suffix_len      // "**" 的长度
  tr.delete(text_end, end)                   // 删除后面的 "**"
  tr.delete(start, text_start)               // 删除前面的 "**"
  tr.add_mark(start, start + text_len, bold) // 给文本添加 bold
  return tr
```

## 与历史的交互

输入规则触发的事务会被标记，以便撤销时能一步还原：

```
用户输入 "## ":
  事务1（被输入规则拦截）: 删除 "## "，段落 → 标题

Ctrl+Z:
  不是撤销标题转换，而是回到输入 "## " 之前的状态
  即还原为包含 "## " 文本的段落
```

这通过在事务上设置元数据实现，历史插件据此正确分组。

---

## 作为插件的实现

```
Plugin:
  props:
    handle_text_input(view, from, to, text):
      let new_text = 光标所在行文本 + text
      for rule in rules:
        if let Some(match) = rule.match.match(new_text):
          let tr = rule.handler(view.state, match, ...)
          view.dispatch(tr)
          return true
      return false
```

输入规则器不持有插件状态——它只是一组正则到 handler 的映射。
