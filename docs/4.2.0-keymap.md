# 快捷键映射器

> 将键盘事件映射为命令函数。按下快捷键时查表，找到对应的命令并执行。

## 原理

快捷键映射器本质是一个 `HashMap<String, Command>`：

```
{
  "Mod-b":       toggle_bold,
  "Mod-i":       toggle_italic,
  "Mod-z":       undo,
  "Mod-Shift-z": redo,
  "Enter":       split_block,
  "Backspace":   delete_backward,
  "Tab":         indent,
  "Shift-Tab":   outdent,
}
```

---

## 键名格式

| 修饰键 | 说明 |
|--------|------|
| Mod | 平台自适应：macOS 上是 Cmd，其他平台是 Ctrl |
| Ctrl | Ctrl 键 |
| Alt | Alt 键（macOS 上是 Option） |
| Shift | Shift 键 |

修饰键之间用 `-` 连接，最后是按键名：`Mod-Shift-Enter`

常见按键名：`Enter`、`Backspace`、`Delete`、`Tab`、`Escape`、`ArrowUp`、`ArrowDown`、`ArrowLeft`、`ArrowRight`、`a`~`z`、`0`~`9`

---

## 命令 (Command)

命令是一个函数：

```
fn command(state: &EditorState) -> Option<Transaction>
```

- 接收当前编辑器状态
- 返回 `Some(tr)` 表示执行成功，事务将被提交
- 返回 `None` 表示该命令在当前状态下不可用（比如在非列表中按 Tab）

### 常用命令

| 命令 | 快捷键 | 说明 |
|------|--------|------|
| toggle_bold | Mod-b | 切换粗体 |
| toggle_italic | Mod-i | 切换斜体 |
| toggle_code | Mod-\` | 切换行内代码 |
| undo | Mod-z | 撤销 |
| redo | Mod-Shift-z | 重做 |
| split_block | Enter | 回车换段 |
| delete_backward | Backspace | 向前删除 |
| delete_forward | Delete | 向后删除 |
| join_backward | Backspace（在块开头） | 与上一个块合并 |
| lift_empty | Enter（在空列表项中） | 提升空列表项为段落 |
| select_all | Mod-a | 全选 |

---

## 多层映射

可以注册多个快捷键映射器，按优先级顺序查找：

```
plugins: [
  keymap(自定义快捷键),      // 优先级最高
  keymap(Markdown专用快捷键), // 其次
  keymap(基础快捷键),         // 兜底
]
```

查找顺序：从第一个映射器开始，找到就执行；找不到就查下一个。命令返回 None 也会继续查下一个（表示当前命令不适用）。

---

## 作为插件的实现

快捷键映射器是一个插件，通过 props 中的键盘事件处理实现：

```
Plugin:
  props:
    handle_key_down(view, event):
      let key_name = event_to_key_name(event)  // "Mod-b"
      if let Some(command) = bindings.get(key_name):
        if let Some(tr) = command(view.state):
          view.dispatch(tr)
          return true  // 已处理
      return false  // 未处理，传给下一个插件
```

不需要自己的插件状态——它只是事件到命令的映射表。
