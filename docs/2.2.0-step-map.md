# 步骤映射器

> 描述单个步骤对文档位置的影响。给定一个变更前的位置，步骤映射器能算出变更后的对应位置。

## 内部结构

步骤映射器存储一组 **替换区间**，每个区间是 (pos, old_size, new_size) 三元组：

- `pos`：替换发生的位置
- `old_size`：被替换的旧内容长度
- `new_size`：替换后的新内容长度

```
删除位置 4~6 的 2 个字符:
  StepMap: [(4, 2, 0)]

在位置 4 插入 3 个字符:
  StepMap: [(4, 0, 3)]

把位置 4~6 的 2 个字符替换为 5 个字符:
  StepMap: [(4, 2, 5)]
```

一个步骤可能产生多个替换区间（比如 ReplaceAroundStep 修改了两处不连续的位置）。

---

## map 方法

```
step_map.map(pos, bias) → new_pos
```

| 参数 | 说明 |
|------|------|
| pos | 变更前的位置 |
| bias | 当位置恰好在插入/删除边界时，偏向哪边。1 = 右（默认），-1 = 左 |

映射规则：

| 位置情况 | 结果 |
|----------|------|
| 在替换区间之前 | 不变 |
| 在替换区间之后 | 偏移 (new_size - old_size) |
| 在替换区间内部 | 映射到替换区间的边界（由 bias 决定左还是右） |

### 完整示例

```
文档: "ABCDEFGH"（位置 0~8）
步骤: 删除 "CD"（位置 2~4）
StepMap: [(2, 2, 0)]

map(0) → 0    在删除区之前，不变
map(1) → 1    在删除区之前，不变
map(2) → 2    恰好在删除区起始，左边界不动
map(3) → 2    在删除区内部，映射到边界（bias=1 → 右边界 = 2）
map(4) → 2    恰好在删除区结束
map(5) → 3    在删除区之后，减 2
map(7) → 5    在删除区之后，减 2
```

### bias 的作用

```
步骤: 在位置 4 插入 "XYZ"（3 个字符）
StepMap: [(4, 0, 3)]

map(4, bias=1)  → 7    光标跟着插入内容走到右边
map(4, bias=-1) → 4    光标留在插入点左边
```

实际使用场景：
- 用户输入文字时，光标用 **bias=1**（跟着文字走）
- 选区的起始端用 **bias=-1**，结束端用 **bias=1**（选区范围自然扩展而不是收缩）

## map_result 方法

```
step_map.map_result(pos, bias) → MapResult { pos, deleted, deleted_across }
```

比 map 返回更多信息：

| 字段 | 说明 |
|------|------|
| pos | 映射后的位置 |
| deleted | 该位置是否被删除操作"吞掉"了（位置在替换区间内部且被替换为不同内容） |
| deleted_across | 该位置的左右两侧内容是否都被删除了 |

`deleted` 信息对装饰器很重要——如果装饰器锚定的位置被删除了，装饰器应该被移除而不是映射到边界。

---

## 关键操作汇总

| 操作 | 说明 |
|------|------|
| StepMap::new(ranges) | 从替换区间数组构造 |
| StepMap::empty() | 无位置变化的空映射 |
| step_map.map(pos, bias) | 映射位置 |
| step_map.map_result(pos, bias) | 映射位置，返回详细结果 |
| step_map.for_each(f) | 遍历所有替换区间 |
| step_map.invert() | 反转映射（交换 old_size 和 new_size） |
