# 装饰器

> 不修改文档的纯视觉叠加。用于搜索高亮、拼写检查标注、评论标记等需要"在文档上画点东西"但不改变文档内容的场景。

## 装饰器类型

### 挂件装饰器 (Widget)

在文档的某个位置插入一个非文档元素。

| 字段 | 说明 |
|------|------|
| pos | 插入位置 |
| side | 在位置的左侧还是右侧（-1 = 左，1 = 右） |
| draw | 绘制函数（在 Canvas 上绘制内容） |

```
在标题旁边显示一个锚点图标:
  Widget { pos: heading 起始位置, side: -1, draw: 绘制 # 图标 }

在段落末尾显示评论数量:
  Widget { pos: paragraph 结束位置, side: 1, draw: 绘制评论气泡 }
```

挂件不占文档位置，不影响位置系统，纯视觉。

---

### 行内装饰器 (Inline)

给范围内的行内内容添加视觉样式。

| 字段 | 说明 |
|------|------|
| from | 范围起始 |
| to | 范围结束 |
| attrs | 样式属性（背景色、边框、自定义绘制等） |

```
搜索高亮 "hello":
  Inline { from: 5, to: 10, attrs: { background: 黄色 } }

拼写错误标注:
  Inline { from: 20, to: 25, attrs: { underline: 红色波浪线 } }
```

行内装饰器和标记 (Mark) 的区别：
- **标记**是文档数据的一部分，持久化保存，参与约束检查
- **行内装饰器**是纯视觉，不存在于文档中，不序列化

---

### 节点装饰器 (Node)

给指定的节点整体添加视觉属性。

| 字段 | 说明 |
|------|------|
| from | 节点起始位置 |
| to | 节点结束位置 |
| attrs | 样式属性（背景色、边框等） |

```
当前光标所在段落的高亮背景:
  Node { from: 4, to: 18, attrs: { background: 浅灰色 } }

拖拽目标节点的蓝色边框:
  Node { from: 20, to: 35, attrs: { border: 蓝色虚线 } }
```

---

## 装饰集 (DecorationSet)

装饰器的持久化集合。不是简单的数组，而是镜像文档树结构的树形数据结构。

### 为什么是树形结构

如果装饰集是一个平铺数组，文档修改后需要遍历所有装饰器更新位置。树形结构使得：

- 局部修改只影响对应子树的装饰器
- 未修改区域的装饰器直接复用，不需要遍历

### 创建

```
DecorationSet::create(doc, decorations):
  将装饰器数组按文档树结构分配到对应的节点子树中
```

### 映射

文档修改后，装饰集需要通过映射器更新位置：

```
new_set = old_set.map(mapping, new_doc)
```

映射时：
- 装饰器的 from/to 通过映射器转换到新位置
- 被删除区域内的装饰器自动移除（根据 map_result.deleted 判断）
- 未受影响区域的装饰器子树直接复用

### 增量更新

```
new_set = old_set.map(mapping, new_doc).add(doc, new_decorations).remove(removed_decorations)
```

| 方法 | 说明 |
|------|------|
| DecorationSet::create(doc, decorations) | 从装饰器数组创建 |
| DecorationSet::empty() | 空装饰集 |
| set.map(mapping, doc) | 映射到新文档 |
| set.add(doc, decorations) | 添加装饰器 |
| set.remove(decorations) | 移除装饰器 |
| set.find(start, end, predicate) | 查找范围内的装饰器 |

## 装饰器在 Canvas 上的绘制

在 cosmic-text 方案下，装饰器的绘制时机和方式：

```
渲染流程:
  1. 绘制块级背景（代码块灰底等）—— 节点自身的样式
  2. 绘制节点装饰器的背景 ← 装饰器
  3. 绘制行内装饰器的背景（搜索高亮等） ← 装饰器
  4. 绘制文本内容（cosmic-text）
  5. 绘制行内装饰器的前景（波浪下划线等） ← 装饰器
  6. 绘制选区高亮
  7. 绘制光标
  8. 绘制挂件装饰器 ← 装饰器
```

## 典型使用场景

### 搜索高亮（插件实现）

```
Plugin:
  state:
    init: DecorationSet::empty()
    apply(tr, set, old_state, new_state):
      if tr.get_meta("search_query") 变了:
        // 重新搜索，创建新的装饰集
        let matches = find_all(new_state.doc, query)
        let decorations = matches.map(|m| Inline(m.from, m.to, {background: 黄色}))
        return DecorationSet::create(new_state.doc, decorations)
      else:
        // 文档修改，映射已有装饰集
        return set.map(tr.mapping, new_state.doc)
```

### 光标所在行高亮（插件实现）

```
Plugin:
  state:
    init: DecorationSet::empty()
    apply(tr, set, old_state, new_state):
      let cursor_block = new_state.selection 所在的块级节点
      let decoration = Node(block.from, block.to, {background: 浅灰})
      return DecorationSet::create(new_state.doc, [decoration])
```
