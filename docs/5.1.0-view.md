# 视图

> 顶层协调者，将编辑器状态渲染到 iced Canvas 上，将用户事件转化为事务。

## 结构

| 字段 | 说明 |
|------|------|
| state | 当前编辑器状态 |
| layout_cache | 块级节点的布局缓存（y 坐标、高度） |
| text_buffer | cosmic-text Buffer，持有排版后的文本数据 |
| scroll_y | 当前滚动位置 |
| viewport_height | 视口高度 |
| decoration_set | 当前的装饰集 |
| is_composing | 是否正在进行输入法组合（IME） |

---

## 核心方法

### dispatch

提交事务，更新状态，触发重渲染：

```
view.dispatch(tr):
  let new_state = self.state.apply(tr)
  self.update_state(new_state)
```

这是视图对外的唯一状态修改入口。所有事件处理最终都调用 dispatch。

### update_state

接收新状态，执行增量更新：

```
view.update_state(new_state):
  let old_state = self.state
  self.state = new_state

  if old_state.doc != new_state.doc:
    // 文档变了，更新布局缓存和 cosmic-text Buffer
    self.update_layout(old_state.doc, new_state.doc, tr.mapping)

  if old_state.selection != new_state.selection:
    // 选区变了，标记光标/选区区域需要重绘
    self.mark_selection_dirty()

  // 更新装饰集
  self.update_decorations(new_state)
```

### update_layout

增量更新布局。利用步骤映射器定位变化区域，只重排受影响的部分：

```
view.update_layout(old_doc, new_doc, mapping):
  // 找到变化范围
  let change_start = mapping 中第一个替换区间的位置
  let change_end = mapping 中最后一个替换区间的结束位置

  // 找到变化范围对应的块级节点
  let first_dirty_block = pos_to_block_index(change_start)
  let last_dirty_block = pos_to_block_index(change_end)

  // 只重新排版这些块的 cosmic-text 内容
  for block in first_dirty_block..=last_dirty_block:
    rebuild_block_buffer(block)

  // 从 first_dirty_block 开始重新计算 y 坐标
  recalculate_y_from(first_dirty_block)
```

## 布局计算

### 块级布局

文档是一系列块级节点纵向排列。每个块有：

| 属性 | 说明 |
|------|------|
| y | 顶部 y 坐标 |
| height | 高度（由 cosmic-text 排版后确定） |
| indent | 左缩进（列表嵌套层级决定） |
| padding | 内边距（代码块、引用块等） |
| margin_top / margin_bottom | 上下间距 |

```
block[0]: heading    y=0,    height=40,  margin_bottom=16
block[1]: paragraph  y=56,   height=24,  margin_bottom=12
block[2]: code_block y=92,   height=80,  padding=8
block[3]: paragraph  y=188,  height=24
```

### 行内布局

行内布局完全交给 cosmic-text 处理：

1. 遍历块级节点内的行内内容
2. 每个文本节点按标记设置 cosmic-text 的属性（字体、字重、大小、颜色）
3. cosmic-text 自动处理换行、字体回退、字形定位
4. 结果：每个字形的精确 (x, y) 坐标

## 坐标转换

### pos_at_coords(x, y) → 文档位置

```
1. y + scroll_y → 找到对应的块级节点（二分查找 layout_cache）
2. x, y 相对于块内偏移 → cosmic-text hit_test → 块内字符偏移
3. 块内字符偏移 → 文档位置（加上块的起始位置）
```

### coords_at_pos(pos) → (x, y)

```
1. pos → 找到所在的块级节点和块内偏移
2. 块内偏移 → cosmic-text 字形位置查询 → (x, y) 相对于块
3. 加上块的 y 坐标 - scroll_y → 屏幕坐标
```

## 光标和选区绘制

| 选区类型 | 绘制方式 |
|----------|---------|
| 光标 | coords_at_pos(pos) 得到坐标，绘制闪烁竖线 |
| 文本选区 | 遍历 from 到 to 的每一行，绘制蓝色背景矩形 |
| 节点选区 | 绘制被选中节点的蓝色边框或背景 |
| 间隙光标 | 在节点间隙位置绘制竖线 |

文本选区可能跨多行、多块，需要逐行计算矩形：

```
选区 from=5 to=30，跨了 3 行:

第 1 行: 从 from 的 x 坐标到行尾 → 矩形1
第 2 行: 整行 → 矩形2
第 3 行: 从行首到 to 的 x 坐标 → 矩形3
```

## IME（输入法）处理

中文/日文等输入法有组合状态（composition），用户在确认之前输入的内容是临时的：

```
1. compositionstart → is_composing = true，记录起始位置
2. compositionupdate → 更新临时文本显示（不提交事务）
3. compositionend → is_composing = false，用最终文本提交事务
```

在 composition 期间不能修改文档（否则会打断输入法状态），只能在 Canvas 上临时绘制组合文本。

## 滚动

| 场景 | 行为 |
|------|------|
| 鼠标滚轮 | 更新 scroll_y，重绘 Canvas |
| 光标移出视口 | 自动滚动使光标可见（scroll_into_view） |
| 页面跳转 | 直接设置 scroll_y 到目标位置 |

scroll_into_view 需要知道光标的屏幕坐标（coords_at_pos），然后调整 scroll_y 使其在视口内。
