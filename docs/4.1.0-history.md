# 历史

> 撤销/重做插件。记录事务的逆向步骤，Ctrl+Z 时倒着应用。利用步骤的可反转性实现精确撤销。

## 原理

每个步骤都可以通过 `step.invert(doc)` 生成逆向步骤。历史插件的工作就是：

1. 每次事务提交时，保存该事务中所有步骤的逆向步骤
2. 撤销时，取出逆向步骤，构造新事务应用

```
用户输入 "abc":
  事务1: insert "a" at 5   → 逆向: delete 5~6
  事务2: insert "b" at 6   → 逆向: delete 6~7
  事务3: insert "c" at 7   → 逆向: delete 7~8

Ctrl+Z (撤销事务3):
  取出逆向步骤: delete 7~8
  构造新事务，应用 → "c" 被删除
```

---

## 插件状态

| 字段 | 说明 |
|------|------|
| done | 撤销栈——已提交事务的逆向步骤集合 |
| undone | 重做栈——已撤销事务的正向步骤集合 |

```
初始: done=[], undone=[]

用户操作 A: done=[A⁻¹], undone=[]
用户操作 B: done=[A⁻¹, B⁻¹], undone=[]
撤销 B:    done=[A⁻¹], undone=[B]
撤销 A:    done=[], undone=[B, A]
重做 A:    done=[A⁻¹], undone=[B]
```

---

## 事务分组

连续的小操作应该合并为一次撤销。比如快速连续输入的字符应该一次 Ctrl+Z 全部撤销，而不是逐字撤销。

分组策略：
- **时间窗口**：短时间内（如 500ms）的连续输入合并为一组
- **显式分隔**：某些操作（回车、删除、粘贴）自动开始新组
- **元数据控制**：事务的 `"addToHistory": false` 标记跳过历史记录

```
快速输入 "hello":
  done 栈中只有一个条目，包含 5 个逆向步骤
  一次 Ctrl+Z 删除整个 "hello"

输入 "hello"，停顿 1 秒，输入 "world":
  done 栈中有两个条目
  第一次 Ctrl+Z 删除 "world"
  第二次 Ctrl+Z 删除 "hello"
```

## 与映射器的交互

撤销时，栈中的逆向步骤可能已经过时（中间有其他操作改变了文档）。需要通过映射器将旧位置转换到当前文档：

```
1. 用户在位置 5 插入 "abc"
2. 用户在位置 20 插入 "xyz"
3. Ctrl+Z 撤销第 2 步

第 2 步的逆向步骤: delete 20~23
这个位置在当前文档中仍然有效（第 2 步是最后一步），直接应用。

但如果撤销第 1 步:
第 1 步的逆向步骤: delete 5~8
中间经过了第 2 步（在位置 20 插入了 3 个字符）
需要映射: 位置 5~8 经过第 2 步的映射 → 仍然是 5~8（在插入点之前，不受影响）
```

## 跳过历史的事务

某些事务不应该被记录：

| 场景 | 说明 |
|------|------|
| 光标移动 | 纯选区变化，没有文档修改 |
| 协同编辑的远端步骤 | 远端操作不应该出现在本地撤销栈 |
| 自动修正 | append_transaction 产生的修正事务 |
| 装饰器更新 | 纯视觉变化 |

通过元数据标记：`tr.set_meta("addToHistory", false)`

---

## 命令

| 命令 | 说明 |
|------|------|
| undo(state) → Option\<Transaction\> | 从 done 栈取出逆向步骤，构造事务 |
| redo(state) → Option\<Transaction\> | 从 undone 栈取出步骤，构造事务 |

命令返回 None 表示栈为空，无法撤销/重做。

---

## 配置

| 参数 | 说明 |
|------|------|
| depth | 最大撤销步数（默认 100） |
| new_group_delay | 连续输入的分组时间窗口（默认 500ms） |
