# 映射器

> 多个步骤映射器的累积。将位置映射经过一整个变换（多个步骤）后得到最终位置。

## 为什么需要映射器

单个步骤映射器只能映射一步。但一个变换通常包含多个步骤，位置需要逐步穿越：

```
步骤1: 删除位置 4~6       → StepMap1: (4, 2, 0)
步骤2: 在位置 10 插入 "X"  → StepMap2: (10, 0, 1)

位置 12 的映射路径:
  12 → 经过 StepMap1 → 10（减 2）
  10 → 经过 StepMap2 → 11（加 1）

最终: 12 → 11
```

映射器把这个过程封装起来，一次调用完成。

---

## 结构

| 字段 | 说明 |
|------|------|
| maps | 步骤映射器数组（按步骤顺序） |
| from | 映射的起始索引（默认 0，用于部分映射） |
| to | 映射的结束索引（默认 maps.len()） |

---

## 关键操作

| 操作 | 说明 |
|------|------|
| Mapping::new() | 创建空映射器 |
| mapping.append_map(step_map) | 追加一个步骤映射器 |
| mapping.append_mapping(other) | 追加另一个映射器的所有步骤映射器 |
| mapping.map(pos, bias) | 将位置映射经过所有步骤 |
| mapping.map_result(pos, bias) | 映射位置，返回详细结果（含 deleted 信息） |
| mapping.slice(from, to) | 取子范围的映射器（只映射部分步骤） |
| mapping.invert() | 反转映射器（所有步骤映射器反转并倒序） |

---

## 与变换器的关系

变换器自动维护一个映射器：

```
let tr = Transform::new(doc);
tr.delete(4, 6);       // 内部: mapping.append_map(步骤1的StepMap)
tr.insert(10, node);   // 内部: mapping.append_map(步骤2的StepMap)

tr.mapping.map(12)     // → 11，自动穿越两个步骤
tr.mapping              // 包含 2 个步骤映射器
```

变换器的后续步骤也会用映射器来转换前面步骤传入的位置——比如用户调用 `tr.delete(4, 6)` 后再调 `tr.insert(10, node)`，变换器内部会先把 10 映射到删除后的位置再执行插入。

---

## 协同编辑中的 rebase

映射器在协同编辑的 rebase 中扮演核心角色。当本地有未确认的步骤，同时收到远端步骤时：

```
本地未确认步骤: [stepA, stepB]
收到远端步骤:   [stepX]

rebase 过程:
1. 用 stepX 的映射 map stepA 中的位置 → stepA'
2. 用 stepX 的映射 + stepA' 的映射 map stepB 中的位置 → stepB'
3. 应用: doc → stepX → stepA' → stepB'
```

映射器的 `slice` 和可组合性使这个过程可以链式处理任意长度的步骤序列。
