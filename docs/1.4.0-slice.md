# 切片

> 文档的局部片段，用于复制粘贴和拖拽。切片记录了内容本身以及边界处的"开放深度"，以便正确融入目标位置。

## 为什么需要切片

用户选中的内容不一定是完整的节点。比如：

```markdown
# 标题

一段**粗体**文本。

> 引用内容
```

如果选中的是从"粗体"到"引用内容"：

```
一段 |**粗体**文本。   ← 选区开始，paragraph 不完整

> 引用内容|            ← 选区结束，blockquote 不完整
```

直接截取会丢失结构信息——粘贴时不知道这些内容原来是在 paragraph 和 blockquote 里面的。切片就是用来保留这个信息的。

---

## 切片组成

| 字段 | 说明 |
|------|------|
| content | 片段 (Fragment)，切片的实际内容 |
| open_start | 开头的开放深度——内容起始侧有多少层节点是"不完整的" |
| open_end | 结尾的开放深度——内容结束侧有多少层节点是"不完整的" |

---

## 开放深度示例

### 完整选中一个段落

```
选中: 整个 "一段粗体文本。" 段落

content: Fragment [paragraph [text("一段"), text("粗体")[bold], text("文本。")]]
open_start: 0    ← paragraph 是完整的
open_end: 0      ← paragraph 是完整的
```

粘贴时：直接作为完整的块级节点插入。

### 选中段落中间一段文字

```
选中: "粗体文"（跨了两个文本节点）

content: Fragment [paragraph [text("粗体")[bold], text("文")]]
open_start: 1    ← paragraph 是开放的（只选了内部一部分）
open_end: 1      ← paragraph 是开放的
```

粘贴时：open_start=1 告诉替换算法"最外层这个 paragraph 不要新建，融入目标位置的 paragraph 里"。

### 跨块级节点选中

```
选中: 从 paragraph 的 "文本。" 到 blockquote 的 "引用内容"

content: Fragment [
    paragraph [text("文本。")],
    blockquote [paragraph [text("引用内容")]]
]
open_start: 1    ← 第一个 paragraph 是开放的
open_end: 2      ← blockquote 开放，里面的 paragraph 也开放
```

---

## 关键操作

| 操作 | 说明 |
|------|------|
| Slice::new(content, open_start, open_end) | 构造切片 |
| Slice::empty() | 空切片单例，用于纯删除操作 |
| Slice::max_open(fragment) | 从片段创建最大开放深度的切片（尽量深地打开两侧） |
| slice.size | content.size |
| slice.cut(from, to) | 截取子切片 |
| slice.eq(other) | 相等比较 |

---

## 使用场景

| 场景 | 说明 |
|------|------|
| 复制粘贴 | 选区 → node.slice(from, to) → 切片 → 序列化到剪贴板；粘贴时反序列化 → node.replace(from, to, slice) |
| 拖拽 | 同上，先 slice 再 replace |
| 替换步骤 | 变换层的 ReplaceStep 核心参数就是 from、to 和一个切片 |
| 删除 | replace(from, to, Slice::empty())——用空切片替换选区 |

切片是连接"用户的选区操作"和"节点树的结构化修改"的桥梁。
